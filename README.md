## 🚀 Featured Project — GitHub RAG Assistant

> *Ask any GitHub repository questions in plain English*

| What it does | How |
|---|---|
| Ingests any public GitHub repo | PyGithub + custom chunker with line numbers |
| Retrieves relevant code chunks | **Hybrid BM25 + ChromaDB** merged via Reciprocal Rank Fusion |
| Re-scores for true relevance | **Cross-encoder reranker** (ms-marco-MiniLM-L-6-v2) |
| Answers in natural language | LLaMA-3.3-70B via Groq, streamed token-by-token |
| Measures quality | **RAGAS evaluation** — faithfulness: 0.38, context recall: 0.10 |

**Why it's not just another RAG demo:** Most RAG projects do vector search → stuff chunks → ask LLM. This one adds hybrid retrieval (catches keyword matches pure vector misses), a reranker (cross-encoder reads query+chunk together for true relevance), and RAGAS evals to prove the upgrade works. Total latency ~1.85s.
